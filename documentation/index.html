3. Input Database information .

4. Input Login details to set login information for this site.

Input all required information shows in figure to complete this step and submit to procced.

Items

Items are the products that you sell to various clients and can be purchased from various vendors in a business. Items are associated with Item categories.
 
Purpose: Add / Edit / Delete/Copy Items of Inventory.
Procedure: Complete item entry is done by five steps.
General settings: 
Item Code:  Not changeable later - unique identifier for an Item.
Name: Name of the Item.
Description: Full/short description of the Item.
Category: The category to which the Item belongs. You can create your own Item categories which define relevant units only.
Item Tax type: Select relevant item tax type.
Item Image: Upload item image in PNG and JPEG format only. 
 
Sales pricing:
Assign sale prices to individual sales Items by sales type. Use them in sales invoices.
Sales type: Add sales type first.
Price: Select price for specific sales type.
Action: Edit only.  
 
Purchase pricing:
Like sales pricing assign purchase prices to individual purchase item for specific supplier. And finally the supplier invoice contains the same info as they receive and the price can be changed, in which case the purchasing data is updated again. 
Supplier: For every Item you can have one or more suppliers for this particular Item (chose from drop down list).
Price: keep entry of purchase price for each item from that specific supplier.    
 
Transaction:
Shows each transaction history of an item with date and quantity.
 
 
 
Status:
When user create an item , it will shows stock status for all location for this company with quantity of item available. 
 
 

Purchase

A purchase order is an official document that you issue to your vendors, indicating relevant information about what you want to buy, the quantity, the price agreed for that particular product or service.

 

Purpose

Purchase Order Entry is used to register all purchase orders in the system. A purchase invoice you create to record a delivery of items made by a vendor to your warehouse. This is your way of recording the items and delivery notes you have received from your vendors for the purchases made by you.

Procedure

  • Select a Supplier;
  • Select a store location;
  • Reference number will generate automatically.
  • select items, supply quantity and click Add Item
  • Write note of purchase order.
  • Submit your order.

 

Permission:

User will have permission to View, Edit and delete purchase order.  User can print, Download PDF, edit or delete invoice.

Sales order:    

A sales order is a financial document that confirms an impending sale. It details the exact quantity, price and delivery details of the products or services being sold. 

 

Purpose
To create Sales orders and thereon to view, modify and print them.
Procedure
The Reference Field is generated automatically. Before a Quote can be created, you must previous have setup one each of the following:
  1. Item
  2. Customer
  3. Customer Branch
  4. Sales type
  5. Payment type
  6. Location

       Then,

  • Select a Customer;
  • Select a store location;
  • Reference number will generate automatically.
  • select items, supply quantity and click Add Item
  • Write note of sales order.
  • Submit your order.

  You can also maintain items & customers in separate browser window but page refresh may be required to update.

 

Notes:Sales order are not related to item stock, user can make sales order for available or empty stock.

Invoice:

An Invoice is a document sent to your client that indicates the products/services sold by you with the payment information that the client has to make. This method allows you to create an invoice directly, by skipping a Sales Order too .Invoice from sales order: Creating invoice from sales order is very easy and simple. User can create both manually and automatically.

                                -       User can create several number of invoice based on item quantity of that sales order.

-       User can create only one invoice automatically from a sales order.

 

New Invoice without sales Order:

The steps and requirements for creating an invoice directly are covered under Sales order entry. By default, this operation will not allow an invoice to be created if the stock level of any item will become less than 0. Invoice entry is directly related to item stock.Creating an invoice entry is almost same like Sales Order entry; just user needs to select a payment term as an affiliation.

 

Notes:

 A direct invoice entry will generate an automatic sales order entry. When entering a direct sales invoice Sales Order will be created automatically.

There are different ways to record payments.

Shipment:

Our powerful shipping integration feature can integrate your organization with a host of shipping service providers, enabling you to get shipping rates, ship your packed goods to your customer, track your shipments and manage your deliveries .

 

User can shipment his product from invoice only.

There are two stages to complete a shipment procedure.

-Packed

-delivered

Packed: When user creates a shipment from an invoice it automatically marked as packed.  User can process shipment both manually and automatically.

Delivered:

혰  You can mark the packages that have been shipped as delivered by clicking on Mark as Delivered button on shipment list. On doing so, your shipment status changes from packed to deliver.

There is an option to view, modify, email, print and download PDF of shipment receipt.

Note: Mark your shipment status delivered after confirming delivery report from your client.  

Customers

This is to manage all customers who are purchasing items/products from this selected company. 

 

There are two different way to create new customer. Admin can create customer by both manually and by uploading CSV files.

For manually adding new customer there is a customer create form which needs customer's name, email , address and phone number .

Admin can import CSV file. There is an option to download customer list in CSV format.

Before Importing one can download a sample CSV file to maintain format of uploading customer list and it is obligatory to input unique email address for each and every individual customer.

Admin has permission of add, edit and delete customers.

Finance:
 Manage finance settings by individually setting bellow types.

Tax:

Define all tax types by mention tax rates.

User can select any tax type as default and user can create as much tax type as they need.

And, user has permission to modify and delete any tax type.

Note: Before using sales tax in your transactions, you need to create a new tax.

Sales Type:

Sales Types allow you to define pricing levels for specific type’s customers. A very common separation is between retail and wholesale customers. 

Here sales type is differ by tax. (Included or not)

Can set any sales type as default according to need.

Currency:

Stockpile is a multicurrency system and it’s simple to manage all the currencies you deal with. Add currency’s with appropriate symbol.

Payment Term:

Set the payment terms for both customers and suppliers. Use them in invoices. Type in a description for the terms you wish to enforce to your customers or have been set by your suppliers. According to payment date/due date user can create several payment terms and use them for making easy his/her payment procedure.

Create a payment term:

Term: name or description for the terms.

Due Day: payment date.

Default: select yes/no to make this payment term default or not.

Example:

Description – net 15
Due Days – 15

Default – No

User can modify or delete any payment term.

Payment Method:

Payment method is nothing but the different ways of paying product cost. There are several ways of payment like- Bank, card, Cash, PayPal etc.

User has permission to modify and delete payment method.
